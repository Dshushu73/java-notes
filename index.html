A Java object stores its state in fields (aka variables) and exposes its behavior through methods (aka functions). Methods operate on an object's internal state and serve as the primary mechanism for object-to-object communication. You can define several methods inside the curly braces for a class. Real-world objects all have state and behavior. Dogs have state (name, color, breed, hungry) and behavior (barking, fetching, wagging tail). Software objects also consist of state and related behavior. Hiding internal data from the outside world, and accessing it only through publicly exposed methods is known as data encapsulation.
